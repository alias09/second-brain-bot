# 🧠 Второй Мозг (Second Brain) — Telegram AI Agent

Автономный ИИ-бот для Telegram, который служит вашим личным ассистентом по ведению базы знаний в Obsidian. Бот построен на архитектуре **Native OpenAI Function Calling**, что делает его полноценным Агентом, способным самостоятельно принимать решения, искать информацию и редактировать файлы.

## ✨ Ключевые возможности

- **Единое контекстное окно:** Больше нет жестких "режимов". Вы просто общаетесь с ботом. Он сам понимает, когда нужно сохранить идею, а когда — найти информацию.
- **Голосовые заметки (Whisper):** Поддержка аудиосообщений. Бот транскрибирует их в текст и работает с ними так же, как с текстовыми.
- **Медиафайлы:** Поддержка отправки фото, видео и документов. Бот сохраняет их в папку `Attachments` и создает markdown-ссылки.
- **Умный семантический поиск:** Бот кэширует вашу базу с помощью эмбеддингов (`text-embedding-3-small`). Он может находить информацию по смыслу, а не только по точным совпадениям.
- **Интеграция с Obsidian и PARA:** Бот автоматически форматирует заметки, добавляет YAML frontmatter (теги, дату, тип) и понимает структуру PARA (Projects, Areas, Resources, Archives).
- **Редактирование существующих заметок:** Бот может находить конкретные файлы (например, список задач) и изменять их (например, заменять чекбокс `- [ ]` на `- [x]`).
- **Мульти-доступ:** Бот отвечает только авторизованным пользователям (ID задаются в конфигурации).

## 🏗 Архитектура

Проект написан на **Python 3.11** с использованием:
- `aiogram` (v3) — для взаимодействия с Telegram API.
- `openai` — для работы с LLM (gpt-4o-mini), эмбеддингами и транскрибацией.
- `numpy` — для вычисления косинусного сходства (cosine similarity) при поиске.
- Фреймворк агента написан с нуля (Custom Agent Loop), что избавляет проект от тяжелых зависимостей вроде LangChain.

### Структура файлов
- `bot/main.py` — Точка входа, инициализация бота и проверка прав доступа (AccessMiddleware).
- `bot/handlers.py` — Обработчики сообщений Telegram (текст, голос, медиа).
- `bot/ai_service.py` — Ядро ИИ-Агента (цикл Tool Calling) и функции транскрибации/векторизации.
- `bot/tools.py` — "Руки" Агента: функции `read_note`, `create_note`, `update_note`, доступные нейросети.
- `bot/search_service.py` — Локальный векторный поиск (RAG) с кэшированием в `.embeddings_cache.json`.
- `bot/obsidian.py` — Прямая работа с файловой системой (сохранение заметок и вложений).

## 🚀 Деплой и Запуск

Бот упакован в **Docker** для бесперебойной фоновой работы.

### Настройка `.env`
В корне проекта должен находиться файл `.env`:
```env
TELEGRAM_BOT_TOKEN="ваш_токен_от_BotFather"
OPENAI_API_KEY="ваш_ключ_openai"
ALLOWED_USER_IDS="12345678, 87654321"
OBSIDIAN_VAULT_PATH="/путь/до/вашего/vault"
```

### Запуск через Docker
```bash
docker-compose up -d --build
```

Бот автоматически подхватывает изменения конфигурации при перезапуске контейнера. Локальная база Obsidian пробрасывается внутрь контейнера (в директорию `/vault`), что позволяет боту безопасно с ней взаимодействовать.
